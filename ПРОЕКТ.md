# 📱 tg_date_walk_bot — Полная документация проекта

---

## 🗺 Что такое этот проект?

Это **Telegram-бот для планирования прогулок и свиданий**.

Представь: ты хочешь погулять с другом или на свидание. Ты уже знаешь любимые места —
кафе, парки, рестораны. Бот берёт эти места, составляет маршрут и отправляет ссылку
на Яндекс Карты. Ты можешь пригласить друга в группу, поделиться маршрутом прямо
в боте, а после прогулки оценить его.

**Простыми словами:** это персональный планировщик прогулок в Telegram.

---

## 🧱 Из чего состоит проект?

Проект делится на три части, которые работают вместе:

```
┌─────────────────┐     HTTP      ┌─────────────────┐
│   Telegram Bot  │ ────────────► │   API (сервер)  │
│  (aiogram 3)    │               │   (FastAPI)     │
└─────────────────┘               └────────┬────────┘
                                           │ SQL
                                  ┌────────▼────────┐
                                  │   База данных   │
                                  │  (PostgreSQL)   │
                                  └─────────────────┘
```

| Часть | Технология | Что делает |
|-------|-----------|-----------|
| **Бот** | Python, aiogram 3 | Общается с пользователем в Telegram |
| **API** | Python, FastAPI | Обрабатывает логику, хранит данные |
| **БД** | PostgreSQL | Хранит места, маршруты, пользователей, группы |
| **Карты** | Яндекс Карты + Nominatim | Геокодирует адреса, строит маршрут |
| **Деплой** | Railway | Сервер, который работает 24/7 без твоего ноутбука |

## 🚀 Как работает бот — пошагово

### Добавление места
1. Пользователь нажимает "➕ Добавить место"
2. Вводит название → адрес → тег (кафе/парк/еда) → ссылку (опционально)
3. Бот определяет координаты через Nominatim (бесплатный геокодер)
4. Место сохраняется в PostgreSQL

### Создание маршрута
1. Пользователь нажимает "🗺 Создать маршрут"
2. Выбирает: только свои места или места из группы
3. Выбирает сценарий: Свидание / Прогулка / Лёгкая прогулка
4. API подбирает места по алгоритму, строит маршрут
5. Бот показывает список мест + ссылку на Яндекс Карты

### Уведомление группы о маршруте
1. Создатель нажимает "🚨 Уведомить участников"
2. Каждый участник группы получает сообщение с маршрутом
3. Участник видит кнопки: **[✅ Пойти]** **[❌ Отклонить]**
4. "Пойти" → создатель получает: "🚶 @username хочет с вами погулять!"
5. "Отклонить" → создатель получает: "😔 @username отказался"
6. После прогулки: "✅ Маршрут пройден" → появляются звёзды [1][2][3][4][5]
7. После оценки → уведомление удаляется из чата

### Группы
- Создать группу → пригласить друга по @username или по ссылке
- Ссылка-приглашение: `https://t.me/БОТ?start=join_ID_ГРУППЫ`
- Друг переходит → бот сразу добавляет в группу
- Можно выйти из группы (с подтверждением)

## ☁️ Деплой на Railway — что это и зачем

**Railway** — это облачный хостинг. Без него бот работает только пока открыт твой ноутбук.
С Railway бот работает 24/7 на удалённом сервере.

### Структура на Railway
Создаётся **3 сервиса** в одном проекте:
1. **api** — запускает FastAPI (railway.json → Dockerfile.api)
2. **bot** — запускает Telegram-бота (railway.bot.json → Dockerfile.bot)
3. **PostgreSQL** — база данных (встроенный плагин Railway)

### Переменные окружения (Railway → Variables)
Для обоих сервисов (api и bot) нужно задать:
```
BOT_TOKEN=токен_от_BotFather
DATABASE_URL=postgresql+asyncpg://...  # из Railway PostgreSQL
API_BASE_URL=http://tgdatewalkbot.railway.internal:8000  # внутренний адрес
BOT_USERNAME=имя_бота_без_@
```

> ⚠️ API_BASE_URL — это **внутренний** адрес Railway, НЕ публичный.
> Формат: `http://ИМЯ_СЕРВИСА.railway.internal:8000`

## 😤 Типовые ошибки и как их решать

### ❌ Ошибка 1: "No start command detected" (Railpack)
**Что случилось:** Railway не знает, как запустить приложение.
**Почему:** Railway по умолчанию пытается угадать команду запуска (Railpack). Если не угадывает — ошибка.
**Решение:** Создать `railway.json` и явно указать Dockerfile:
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {"builder": "DOCKERFILE", "dockerfilePath": "Dockerfile.api"},
  "deploy": {"startCommand": "uvicorn api.main:app --host 0.0.0.0 --port 8000"}
}
```

---

### ❌ Ошибка 2: Бот запускает API (и наоборот)
**Что случилось:** Оба сервиса использовали один и тот же `railway.json` → оба запускали API.
**Почему:** У бота не был указан отдельный конфиг.
**Решение:**
1. Создать `railway.bot.json` с `"dockerfilePath": "Dockerfile.bot"`
2. В Railway → Settings сервиса бота → поле **Config file path** → указать `railway.bot.json`

---

### ❌ Ошибка 3: Docker кэширует старые слои (изменения не применяются)
**Что случилось:** Пушим изменения → Railway деплоит → но работает старый код.
**Почему:** Docker кэширует шаг `COPY . .`. Если хэш файлов совпадает (из-за кэша Railway) — шаг не перевыполняется.
**Решение:** Добавить в Dockerfile строку, которая всегда меняется:
```dockerfile
ENV APP_VERSION=v3-bust-cache
```
Или изменить что-то выше `COPY . .` в Dockerfile, чтобы сбить кэш.

---

### ❌ Ошибка 4: Таблицы не создались в базе данных
**Что случилось:** Бот/API отвечает ошибкой 500, в логах `UndefinedTableError`.
**Почему:** Миграции Alembic не были запущены на Railway.
**Решение:** Зайти в Railway → PostgreSQL → Query и выполнить SQL вручную
(каждый запрос отдельно — Railway Query UI принимает только по одному):
```sql
CREATE TABLE users (...);
CREATE TABLE places (...);
-- и т.д.
```
> ⚠️ Railway Query UI выполняет ТОЛЬКО ОДИН запрос за раз. Несколько через `;` — не работает.

---

### ❌ Ошибка 5: PostGIS / колонка geom не создаётся
**Что случилось:** `UndefinedColumnError: column geom does not exist`
**Почему:** Railway PostgreSQL не поддерживает PostGIS нормально, расширение либо не установлено, либо не работает как надо.
**Решение:** Полностью убрать PostGIS из проекта:
- Удалить `geoalchemy2` и `shapely` из `requirements.api.txt`
- Убрать `geom` колонку из `db/models.py`
- Оставить только `lat: Float` и `lon: Float`

---

### ❌ Ошибка 6: Бот не отвечает при закрытом ноутбуке
**Что случилось:** Бот работает локально И на Railway одновременно → конфликт.
**Ошибка:** `TelegramConflictError: terminated by other getUpdates request`
**Почему:** Два экземпляра бота не могут одновременно получать обновления от Telegram.
**Решение:** Остановить локальный бот:
```bash
docker compose down
```
Или просто закрыть терминал, где запущен бот.

---

### ❌ Ошибка 7: Кнопка зависает (не реагирует на нажатие)
**Что случилось:** Пользователь жмёт кнопку — крутится спиннер, ничего не происходит.
**Почему:** В aiogram, если `callback_query` не вызвал `call.answer()` в течение 10 секунд —
Telegram показывает "зависание". Это происходит когда:
- В коде до `call.answer()` возникла ошибка
- `edit_text()` с неверным `parse_mode` падает с исключением

**Конкретный пример:** Кнопка "Маршрут пройден" пыталась:
```python
await call.message.edit_text(
    call.message.text + "⭐ Оцените маршрут",  # ← проблема здесь
    parse_mode="Markdown",
)
```
`call.message.text` возвращает текст БЕЗ markdown-символов. Telegram получал
невалидную Markdown-разметку → отклонял → `call.answer()` не вызывался → зависание.

**Решение:** Использовать `edit_reply_markup()` вместо `edit_text()` — менять только кнопки:
```python
await call.message.edit_reply_markup(reply_markup=новая_клавиатура)
await call.answer("Подсказка для пользователя")
```

---

### ❌ Ошибка 8: Уведомления поднимают "пункт управления" вверх
**Что случилось:** После отправки маршрута группе бот отправлял новое сообщение
"✅ Маршрут отправлен". Это сообщение появлялось ниже основного меню, из-за чего
меню уезжало вверх и пользователь должен был скроллить вниз.

**Решение:** Вместо `call.message.reply()` (создаёт новое сообщение) →
использовать `call.answer(show_alert=True)` (всплывающий попап):
```python
# ❌ Было (создаёт новое сообщение в чате):
await call.message.reply("✅ Маршрут отправлен!")

# ✅ Стало (показывает попап, чат не меняется):
await call.answer("✅ Маршрут отправлен!", show_alert=True)
```

---

### ❌ Ошибка 9: API_BASE_URL указывает на локальный адрес
**Что случилось:** Бот не может достучаться до API, ошибка 500.
**Почему:** В переменных Railway был указан `http://localhost:8000` или внешний URL.
**Решение:** Использовать **внутренний** Railway-адрес:
```
API_BASE_URL=http://tgdatewalkbot.railway.internal:8000
```
Внутренние адреса Railway работают только между сервисами одного проекта
и имеют формат: `http://ИМЯ_СЕРВИСА.railway.internal:ПОРТ`

## 🛠 Как поднять проект с нуля

### Шаг 1: Получить токен бота
1. Открыть Telegram, найти `@BotFather`
2. Отправить `/newbot`
3. Придумать имя и username (username должен заканчиваться на `bot`)
4. Сохранить токен вида `1234567890:ABCDE...`

### Шаг 2: Создать проект на Railway
1. Зайти на [railway.app](https://railway.app)
2. New Project → Deploy from GitHub repo → выбрать репозиторий
3. Добавить плагин PostgreSQL: `+ New` → Database → PostgreSQL

### Шаг 3: Настроить сервис API
1. Кликнуть на сервис → Settings
2. **Build** → Dockerfile Path: `Dockerfile.api`
3. **Config file path**: `railway.json`
4. Variables → добавить:
   ```
   BOT_TOKEN=твой_токен
   DATABASE_URL=postgresql+asyncpg://postgres:ПАРОЛЬ@postgres.railway.internal:5432/railway
   ```

### Шаг 4: Создать сервис бота
1. `+ New` → GitHub Repo → тот же репозиторий
2. Settings → **Config file path**: `railway.bot.json`
3. Variables → добавить:
   ```
   BOT_TOKEN=твой_токен
   DATABASE_URL=postgresql+asyncpg://postgres:ПАРОЛЬ@postgres.railway.internal:5432/railway
   API_BASE_URL=http://ИМЯ_API_СЕРВИСА.railway.internal:8000
   BOT_USERNAME=имя_бота_без_@
   ```

> Пароль от PostgreSQL: кликнуть на сервис PostgreSQL → Variables → `POSTGRES_PASSWORD`

### Шаг 5: Создать таблицы в базе данных
Зайти в PostgreSQL → Query и выполнить каждый `CREATE TABLE` из `alembic/versions/` по отдельности.

### Шаг 6: Задеплоить
Railway деплоит автоматически при каждом `git push` в main.

Если не задеплоился → зайти в сервис → Deployments → Deploy (вручную).

## 📌 Важные мелочи, которые легко забыть

1. **`call.answer()` обязателен** в каждом callback-handler — иначе кнопка зависнет.
   Вызывать в течение 10 секунд после нажатия.

2. **Railway Query UI** — только один SQL-запрос за раз.

3. **Два экземпляра бота** не могут работать одновременно. Если запускаешь локально —
   останови Railway-сервис (и наоборот).

4. **`call.message.text`** возвращает текст без markdown. Не передавай его обратно
   в `edit_text(..., parse_mode="Markdown")` — получишь ошибку.

5. **Internal URL Railway** работает только между сервисами одного проекта.
   Из интернета по нему не достучаться — это нормально.

6. **BOT_USERNAME** нужен обоим сервисам (api и bot) для генерации ссылок-приглашений.
